status active

Cluster-specific archive. Files moved here are kept for retrospective queries but no longer compete for daily attention.

What goes here

  • Superseded versions of cluster docs (e.g., COLOR_CODING_SYSTEM_v0.md once v1 is locked)
  • Closed-position docs whose canonical _index.md has status: closed
  • Old design specs replaced by newer iterations
  • Quarterly snapshots that have rolled past the active window

What does NOT go here

  • Auto-emitted artifacts (those live in /00_SYSTEM/EXTRACTION/, event-alerts/, etc.)
  • Active drafts (use #status/draft instead)
  • Files you might still need to edit (move them only when fully superseded)

Visual

Per COLOR_CODING_SYSTEM_v1 §3 — anything in any _ARCHIVE/ folder shows as dark gray in the graph view. The graph rule is path:_ARCHIVE (matches all six cluster archives).
